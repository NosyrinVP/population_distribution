# Пространственное распределение региональных статистических данных по территории субъекта РФ на сетке 1×1 км без метода усреднения

## Описание проекта

Проект реализует пайплайн для пространственного распределения населения внутри муниципалитетов на регулярную сетку 1×1 км.
Так как реальные данные о населении на уровне отдельных grid-клеток отсутствуют, задача решается через построение синтетического таргета на основе пространственных признаков: застройки, дорожной сети, ночных огней, природных территорий и рельефа.

Итоговая модель оценивает долю населения каждой grid-клетки внутри муниципалитета, после чего агрегированное население муниципалитета распределяется между клетками пропорционально предсказанным долям.

---

## Структура пайплайна

Пайплайн состоит из двух основных частей:

1. Подготовка пространственных признаков.
2. Обучение моделей распределения населения.

---

## 1. Подготовка данных

На первом этапе создается регулярная сетка 1×1 км и для каждой клетки рассчитываются пространственные признаки.

### Входные данные

Для каждого региона используются следующие слои:

* `admin` — границы муниципалитетов;
* `buildings` — здания;
* `roads` — дорожная сеть;
* `nature` — природные территории;
* `ntl_path` — растровый слой ночных огней;
* `relief_folder` — папка с растровыми данными рельефа;
* `population_df` — таблица населения по муниципалитетам.

### Основные этапы подготовки

1. Все векторные слои приводятся к единой метрической проекции.
2. Создается регулярная сетка с размером ячейки 1000×1000 метров.
3. Каждая клетка привязывается к муниципалитету по максимальной площади пересечения.
4. Для каждой клетки рассчитываются признаки застройки:

   * суммарная площадь зданий;
   * количество зданий;
   * доля застроенности.
5. Рассчитываются признаки дорожной сети:

   * длина дорог;
   * плотность дорог.
6. Рассчитываются признаки природных территорий:

   * площадь природных объектов;
   * доля природных территорий.
7. По растровому слою ночных огней считаются:

   * среднее значение;
   * сумма;
   * максимум.
8. По данным рельефа считаются:

   * средняя высота;
   * минимальная высота;
   * максимальная высота;
   * стандартное отклонение;
   * диапазон высот.
9. Население муниципалитетов присоединяется к grid-клеткам.
10. Итоговая сетка сохраняется в формате GeoPackage.

### Выход подготовительного этапа

Результатом является файл:

```text
grid_1km_<region>_v11.gpkg
```

Для объединения нескольких регионов формируется общий файл:

```text
grid_1km_Combined_v11.gpkg
```

---

## 2. Постановка задачи моделирования

Исходные данные содержат население только на уровне муниципалитета, но не на уровне отдельных grid-клеток. Поэтому прямой supervised learning невозможен.

Для решения задачи создается синтетический таргет — proxy-распределение населения внутри муниципалитета.

---

## 3. Формирование proxy target

Для каждой grid-клетки рассчитывается эвристический spatial score:

```text
proxy_score =
0.35 * log(1 + total_building_area)
+ 0.25 * log(1 + number_of_buildings)
+ 0.20 * log(1 + road_length)
+ 0.20 * log(1 + ntl_mean)
- 0.30 * nature_density
```

Смысл proxy score: клетки с большей застройкой, большим количеством зданий, развитой дорожной сетью и более высокими ночными огнями считаются более вероятными местами концентрации населения. Природные территории, наоборот, уменьшают вероятность размещения населения.

Пустые клетки, где отсутствуют здания, дороги и значимая застройка, получают очень низкий score.

Затем внутри каждого муниципалитета score преобразуется в долю населения с помощью softmax:

```text
proxy_share = softmax(proxy_score)
```

После этого население муниципалитета распределяется по клеткам:

```text
proxy_population = proxy_share * total_population
```

---

## 4. Зачем используется смешивание proxy и baseline

Proxy-распределение может быть слишком агрессивным: большая часть населения может концентрироваться в нескольких клетках, особенно если данные о зданиях, дорогах или ночных огнях неполные.

Чтобы сделать таргет более устойчивым, используется равномерный baseline:

```text
uniform_share = 1 / number_of_cells_in_municipality
```

Затем proxy-распределение смешивается с равномерным распределением:

```text
mixed_share = alpha * proxy_share + (1 - alpha) * uniform_share
```

В проекте используется:

```text
alpha = 0.85
```

Это означает, что модель на 85% опирается на пространственный proxy и на 15% — на равномерное распределение.

Такой подход сглаживает экстремальные значения, снижает риск переобучения на шумный proxy target и не позволяет модели полностью обнулять периферийные клетки.

Итоговый обучающий таргет:

```text
log_proxy_share = log(mixed_share + eps)
```

Логарифмирование используется для уменьшения скошенности распределения и более стабильного обучения модели.

---

## 5. Используемые признаки модели

В модель передаются базовые spatial-признаки:

* площадь муниципалитета в клетке;
* площадь зданий;
* количество зданий;
* доля застроенности;
* длина дорог;
* плотность дорог;
* площадь природных территорий;
* доля природных территорий;
* признаки ночных огней;
* признаки рельефа.

Также создаются дополнительные engineered features:

* логарифмы площади зданий, числа зданий, длины дорог, природных территорий и ночных огней;
* взаимодействие застройки и дорог;
* взаимодействие застройки и ночных огней;
* взаимодействие дорог и ночных огней;
* отношение площади зданий к длине дорог;
* отношение застроенности к природным территориям;
* штраф за природные территории;
* показатель неровности рельефа.

---

## 6. Regression model

Первая модель — `CatBoostRegressor`.

Она обучается предсказывать логарифм доли населения клетки:

```text
target = log(mixed_share + eps)
```

После предсказания значение переводится обратно из log-scale:

```text
pred_score = exp(pred_log_share_raw)
```

Затем предсказания нормализуются внутри каждого муниципалитета:

```text
pred_share = pred_score / sum(pred_score)
```

Итоговое население клетки:

```text
pred_population = pred_share * total_population
```

### Метрики regression model

```text
RMSE log-share: 0.464
MAE share: 0.00030
R2 log-share: 0.801
Correlation share: 0.974
Mean municipality share sum: 1.0
```

### Интерпретация

Regression-модель хорошо воспроизводит синтетический таргет: R² = 0.801 показывает, что модель объясняет около 80% вариации log-share, а корреляция 0.974 говорит о сильном совпадении между proxy-распределением и предсказанными долями. Сумма долей внутри муниципалитетов равна 1, значит условие сохранения общего населения выполняется.

---

## 7. Ranking model

Вторая модель — `CatBoostRanker`.

Она решает не задачу точного предсказания доли, а задачу ранжирования клеток внутри муниципалитета по относительной важности.

Для ranking-модели создается relevance target:

```text
relevance = log1p(proxy_share / mean(proxy_share))
```

Чем выше relevance, тем более приоритетной считается клетка внутри муниципалитета.

Модель обучается с функцией потерь:

```text
YetiRank
```

и метрикой:

```text
NDCG@20
```

NDCG показывает, насколько хорошо модель располагает наиболее важные клетки в верхней части ранжированного списка. Значение 1.0 соответствует идеальному порядку.

### Метрики ranking model

```text
NDCG@20: 0.983
RMSE share: 0.00277
MAE share: 0.00054
R2 share: 0.692
Correlation share: 0.924
Mean municipality share sum: 1.0
```

### Интерпретация

Ranking-модель почти идеально определяет наиболее важные клетки внутри муниципалитетов: NDCG@20 = 0.983 означает очень высокое качество ранжирования top-20 клеток. При этом R² и корреляция ниже, чем у regression-модели, потому что ranking-модель лучше восстанавливает относительный порядок клеток, чем точные значения долей населения.

---

## 8. Hybrid model

Финальная модель объединяет результаты regression и ranking подходов.

Используется взвешенное среднее:

```text
hybrid_share = beta * pred_share + (1 - beta) * rank_share
```

В проекте используется:

```text
beta = 0.7
```

Это означает, что 70% веса получает regression-модель и 30% — ranking-модель.

После смешивания доли снова нормализуются внутри каждого муниципалитета:

```text
hybrid_share = hybrid_share / sum(hybrid_share)
```

Итоговое население клетки:

```text
hybrid_population = hybrid_share * total_population
```

### Интерпретация

Гибридная модель объединяет сильные стороны двух подходов: regression-модель лучше оценивает абсолютные значения долей, а ranking-модель лучше выделяет наиболее приоритетные клетки. Итоговое распределение становится более устойчивым и сбалансированным для применения на новых территориях.

---

## 9. Проверка на Курской области

После обучения на данных Брянской области модель применяется к Курской области.

Для Курской области выполняются те же этапы:

1. загрузка подготовленной grid-сетки;
2. расчет engineered features;
3. предсказание regression score;
4. предсказание ranking score;
5. нормализация долей внутри муниципалитетов;
6. расчет hybrid_share;
7. расчет hybrid_population;
8. визуализация итогового распределения.

Итоговая карта показывает пространственное распределение населения по grid-клеткам на основе гибридной модели.

## 11. Основные ограничения

Метрики модели показывают качество воспроизведения синтетического proxy target, а не точность относительно реального населения на уровне grid-клеток.

Это значит, что модель хорошо научилась имитировать заданную эвристику распределения населения, но для проверки реальной точности необходим внешний источник валидации, например:

* данные переписи на более детальном уровне;
* данные населенных пунктов;
* WorldPop;
* GHSL Population Grid;
* Meta High Resolution Population Density;
* адресные данные;
* данные мобильной активности.

---

## 12. Общий вывод

Разработанный пайплайн позволяет перейти от агрегированного населения муниципалитетов к детальному распределению населения по сетке 1×1 км. Модель использует spatial-признаки территории и комбинирует regression и ranking подходы, что позволяет получить более устойчивое и интерпретируемое распределение населения внутри муниципалитетов.

Regression-модель хорошо восстанавливает величины долей населения, ranking-модель эффективно определяет наиболее приоритетные клетки, а гибридный подход объединяет оба результата в итоговую карту распределения населения.
